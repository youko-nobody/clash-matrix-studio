# Clash Matrix Studio V5.3

Clash Matrix Studio V5.3 是一个基于 Cloudflare Workers 的 Clash/Meta 订阅管理与节点可视化编辑面板。

它把传统“订阅转换器”升级成了可视化节点工作台，支持节点提取、重命名、排序、删除、分组编排、规则编辑和最终订阅输出。

## 项目简介

本项目面向需要精细管理 Clash 节点的用户。它可以把机场订阅、单节点链接和手动积木节点统一解析为可编辑节点块，再通过面板进行双向同步修改，最终输出标准 Clash 配置文件。

核心特点：
- Cloudflare Workers 单文件部署
- Cloudflare KV 持久化存储
- 节点积木式编辑
- 左右双向同步排序
- 规则组可视化编辑
- 实时内存编译预览
- `/sub` 标准订阅输出

## 项目优势

### 1. 单节点级精细控制
不是简单地做订阅转换，而是把每个节点拆成独立模块，方便单独改名、删改、排序和溯源。

### 2. 双向同步编辑
左侧个人节点库和右侧全局拓扑列表实时联动，拖拽排序后会同步更新最终输出。

### 3. 可视化规则编排
支持自定义策略组、分流规则和 `url-test` 自动测速组，适合搭建个人专属配置。

### 4. 云端持久化
使用 Cloudflare KV 保存个人节点库、节点顺序、删除状态、分流规则和可视化配置。

### 5. 单文件易开源
整套逻辑集中在一个 `js` 文件中，便于阅读、部署、维护和二次开发。

### 6. 自动容错
对空数据、脏数据、Base64 乱码和部分异常输入做了自愈处理，降低面板崩坏概率。

## 部署方式

### 1. 准备 Cloudflare KV

在 Cloudflare Dashboard 中创建一个 KV Namespace，例如：

- `CLASH_MATRIX_DB`

### 2. 创建 Worker

新建一个 Cloudflare Worker，例如：

- `clash-matrix-panel`

### 3. 绑定环境变量

在 Worker 设置里添加：

- `CLASH_KV` 绑定到上面的 KV Namespace
- `ADMIN_USER` 管理员账号
- `ADMIN_PASS` 管理员密码

如果不设置，默认账号密码都是 `admin`

### 4. 部署代码

把 `Clash Matrix Studio V5.3.js` 的内容粘贴到 Worker 编辑器，保存并部署。

### 5. 访问面板

部署后打开 Worker 域名，例如：

- `https://xxx.workers.dev/panel`

订阅地址：

- `https://xxx.workers.dev/sub`

## 使用教程

### 1. 导入订阅

在左侧输入订阅链接，点击解析。

系统会把订阅拆成节点块并显示在列表中。

### 2. 提取节点

在解析结果里选择你要保留的节点，加入个人节点库。

### 3. 编辑节点

可以直接修改节点名、调整顺序、删除不需要的节点，或者恢复黑名单节点。

### 4. 配置规则组

在右侧规则编辑区修改分流规则、策略组和测速组。

### 5. 实时预览

点击“触发内存编译输出测试”，查看当前配置生成结果。

### 6. 保存生效

点击“全局固化并部署生效”，把当前状态写入 KV。

### 7. 客户端订阅

把页面顶部的 `/sub` 链接复制到 Clash、Stash、FlClash 等客户端里更新即可。

## 常见问题

### 1. 为什么我改了测速地址，输出没变？

通常是 KV 里还保存着旧的 `clash_rules`。需要重新保存一次面板配置，或者手动更新数据库里的规则内容。

### 2. 为什么面板看起来和 `/sub` 不一样？

面板展示的是当前内存状态，`/sub` 是最终订阅输出。两者应在保存后保持一致。

### 3. 会不会 DNS 泄露？

这取决于你的 DNS 设置、客户端和使用方式。当前项目可以编译出 `dns` 配置，但浏览器 WebRTC 和客户端本地策略仍需要额外处理。
